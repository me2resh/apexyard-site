02Install & open it the first time

ApexDock isn't distributed through the App Store, so macOS will block the first launch until you approve it once. This is normal for a direct download — here's the full flow.

  1. Open the .dmg and drag ApexDock into Applications

    Double-click the downloaded ApexDock-0.8.0.dmg, then drag the ApexDock icon onto the Applications folder.

  2. First open is blocked — click Done (not “Move to Trash”)

    Open ApexDock from Applications. macOS shows “ApexDock Not Opened.” This is Gatekeeper, not an error. Click Done — do not click “Move to Trash.”

  3. Approve it in Privacy & Security → Open Anyway

    Open System Settings → Privacy & Security, scroll to the Security section. You'll see “ApexDock was blocked to protect your Mac.” Click Open Anyway.

  4. Confirm Open Anyway

    macOS asks once more — click Open Anyway.

  5. Enter your Mac password

    Authorize with your administrator password. ApexDock opens — and it won't ask again; every future launch is a normal double-click.

Why the warning? ApexDock is a direct download rather than an App Store / notarized build, so macOS can't auto-verify the developer. You're approving it once, deliberately, from a source you trust — after that it runs like any other app.

03Using ApexDock

Each tile is a launcher — an app to open, a shell command or script to run, or a URL to visit. Click Launch to run it. Click a tile to select it — the right-hand pane shows that launcher's live console and its run history, so you can see what happened (and why something failed) instead of guessing.

＋ Add
Add any launcher — open an app, run a shell command / multi-step script, or open a URL. A guided form fills in the details so every launcher is valid (you can't accidentally break the config).
✦ AI
Describe a launcher in plain English and let ApexDock build it for you.
Live console
Every run streams into the right-hand pane — with search, copy, and Stop / Re-run, plus the full history of past runs. Toggle the pane with the ▤ button.
Bookmarks & apps
Pin your sites, folders, and apps to the top bar for instant access.
Undo / Redo
⌥⌘Z / ⇧⌥⌘Z — every add / edit / remove is reversible.
Themes
Light & dark, follow-system or pinned.

Editing launchers

Use the ＋ Add form — it's the safe path. If you ever edit the config file directly (~/Library/Application Support/ApexDock/launcher-items.yaml), quit ApexDock first (it rewrites the file on quit) and keep every item's action: block.
